Introduction

"Our App: Multi Purpose Phonegap App with Admin Panel, Website & Rest Api" can be used for various types of project applications. Various niche based mobile app projects for like CRM, Freelancing, Blogging, Ecommerce, University, Hospital, Social Media and General make this app a complete one stop solution for all your needs.

Phonegap App is designed using Materialize CSS Framework. It is modern responsive front-end framework based on Material Design. It is specially designed to give your mobile app a unique and elegant look. It is easy to customize and coded in developer friendly manner.

Our App App Admin Panel is a premium admin dashboard built with the Powerful React framework and Bootstrap 4 with Reactstrap and create-react-app. It is the backend of our project. It gives full access to administrator of phonegap app. Our App App Admin Panel includes many react based plugins.

Our App Rest Api helps keeping all our app logic and store data in database. Rest Api makes our app content dynamic. It connects Phonegap app and backend panel together. We use PHP as our backend programming language and MySql as database for Rest Api.

Let's Get Started with Installation...

System Requirements

Before proceeding further with installation we need to make sure we have all the required softwares and applications installed on our system.

You will need to have installed Node.js on your computer. Also insure that you have NPM installed.
You can check if these are installed properly by opening up a Terminal (Linux or Mac) or Command Prompt (Windows)
and entering the command: node --version and npm --version These commands should output something similar to "v12.18.0" and "6.14.4" respectively.

Installing Cordova

Open up a Terminal (Linux or Mac) or Command Prompt (Windows) and Run below command to install cordova globally on your system 

npm install -g cordova

More info about Cordova can be found here: https://cordova.apache.org/#getstarted

Installing Phonegap

Open up a Terminal (Linux or Mac) or Command Prompt (Windows) and Run below command to install phonegap globally on your system 

npm install -g phonegap@latest

More info about Phonegap can be found here: http://docs.phonegap.com/

Installing Android Studio

You will need Android studio on your system to build android version of this app.

Download and Install Android Studio from here: https://developer.android.com/studio/

Check Other Requirements

To make things working smoothly, just make sure you have jdk 1.8.x version installed on your machine.

Also make sure that JAVA_HOME enviornment variable is set in your system.

More info about Java can be found here: https://www.oracle.com/in/java/technologies/javase-downloads.html

Also please make sure you have gradle installed on your system.

More info about Gradle and its installation can be found here: https://gradle.org/install/

To know more about the these requirements visit: https://cordova.apache.org/docs/en/latest/guide/platforms/android/#installing-the-requirements.

Create & Configure Database

First, unzip the downloaded package.

You can choose to install this app on a local server or on a live website.

If you choose to install this app locally, you must install a WAMP Server or XAMPP Server.

Open your server phpmyadmin and create a new database eg: Our App-database

Create a database user and assign it to your newly created database

Assume that your database user has username: admin and password: admin

Now open the unzipped downloaded project folder and open Our App-rest-api/lib/db.php file in your favorite editor and enter your database related details like your host server, database name, database user and database password as shown below

$db['serverhost'] = 'localhost';
$db['serverdatabase'] = 'Our App-database';
$db['serveruser'] = 'admin';
$db['serverpassword'] = 'admin';

Note: Please create a strong password for your database user

If all seems good, proceed to next step

Rest Api

Now lets import the database tables of our project.

From your downloaded package, open database folder. You will find a file rest-api-database.sql. Import this file in your database (created in above step).

Create a rest api folder on your server as eg: /Our App-phonegap/rest-api

Now copy the contents of Our App-rest-api (in downloaded package) and upload it on your server in /Our App-phonegap/rest-api folder. Lets assume that you upload it on following path http://mydomain.com/Our App-phonegap/rest-api/

Your lib/db.php file should be accessible from following path: http://mydomain.com/Our App-phonegap/rest-api/lib/db.php

Note: Along with creating all necessary tables in database, This step will create a new user row in database. This user will be a admin user with credentials username: admin and password: admin. This info will be used in steps while configuring admin panel later.

If you followed all above steps correctly then it should finish your rest api installation for this project.

Create Cordova/Phonegap App

Go to the directory where you maintain your source code, and you can create a cordova project by running the following command in your terminal

cordova create foldername com.phonegap.name appname

To install our Our App-phonegap project run the following command. Make the changes as your requirements:

cordova create Our App-phonegap com.phonegap.Our App Our App

This creates the required directory structure for your cordova app. By default, the cordova create script generates a skeletal web-based application

Now navigate to your newly created project folder and replace the files with our project files

Copy files from downloaded project Our App-phonegap folder and paste them in your newly created cordova app folder. Please take care of maintaing the proper directory structure while replacing the files.

Configure App

Before proceeding further, we need to make sure we connect our Rest Api (configured earlier) with this app.

Open Our App-phonegap/www/assets/js/variables.js file in your favorite editor.

Replace the following line with var REST_API_URL = 'https://mydomain.com/Our App-rest-api/' with your Rest Api url

If your rest api is installed as per the steps mentioned above, then your replace it as var REST_API_URL = 'https://mydomain.com/Our App-phonegap/rest-api/'

Please make sure to add slash (/) at end of url above.

This should connect your rest api with your phonegap app.

Add Platforms

All subsequent commands need to be run within the project's directory

Navigate to your project directory

cd Our App-phonegap

Add the platforms that you want to target your app. We will add the 'ios' and 'android' platform and ensure they get saved to config.xml and package.json:

cordova platform add ios 
cordova platform add android

Running commands to add or remove platforms affects the contents of the project's platforms directory, where each specified platform appears as a subdirectory.

Admin Panel

Now its time to set up our admin panel / backend for this app.

Now open the unzipped downloaded project folder and copy Our App-reactjs-admin folder to your desired location.

Then open your terminal window

Please note that any commands must be ran from the project's root folder.

Open the your project folder in the terminal as per below instructions

On the Mac you can just type 

cd

into the command line and then drag and drop the project folder on to the terminal window. This will add the path of the project directory next to the cd you just typed. Hit return and you have changed to your project folder.

On Windows you will have to open cmd.exe and then type in 

cd path\to\project\directory

and hit return to change into your project folder.

Installing plugins

From the project's root folder type 

npm install

into the command line.

This will install all the necessary packages for this admin panel. You will notice a newly create node_modules folder in your project root directory

Configure Admin Panel

Before proceeding further, we need to make sure we connect our Rest Api (configured earlier) with this app.

Open Our App-reactjs-admin/.env file in your favorite editor.

Replace the below code with your values

REACT_APP_BASEDIR=/Our App-phonegap/admin
REACT_APP_IMGDIR=/Our App-phonegap/admin
REACT_APP_URL=https://mydomain.com/Our App-phonegap/admin
REACT_APP_RESTAPI=https://mydomain.com/Our App-phonegap/rest-api/

According to above configuration your app will be hosted at https://mydomain.com/Our App-phonegap/admin and your Rest Api url is https://mydomain.com/Our App-phonegap/rest-api/

Replace the values as per your requirements

Also note that REACT_APP_BASEDIR and REACT_APP_IMGDIR are same in values.

REACT_APP_URL is made of REACT_APP_BASEDIR at its end.

Please make sure to add slash (/)of url above (at end rest api url only).

Open Our App-reactjs-admin/.env.production file in your favorite editor and copy the contents on .env file as it is. You can read more about .env.production files in reactjs from their official site.

This should connect your rest api with your admin panel.

Run Admin Panel

We've made it easy to develop and test on your local machine. You can start a local server by running the command

From the project's root folder type 

npm start

into the command line.

This will start a local server and open up your browser.

Then you can start your browser and visit this link

http://localhost:3000/

You should be redirected to login screen. For first access in admin panel you can use credentials as username: admin and password: admin. You can change password after login.

App Google Login

This phonegap app has an option to login user with a google account.

You need to create your own Oauth Client ID from Google Developer Console before making the google login work properly in this app.

First login in your google console developer here https://console.developers.google.com/

Create new project or choose existing one.

Go to library and search google+ api. Click Enable on google+api page.

Then go to credentials in side menu and click on configure consent screen button. On next screen, select user type as external and click create.

On next page of Oauth Content Screen Fill application name and select your email. Click Save.

Then Go to credentials in side menu and click create credentials button. Select oauth client id

On next page, select application type as android. You can choose your own platform of choice as well. For this documentation purpose we have included this process for android platform.

Then go to project root directory in terminal and run this

keytool -exportcert -keystore C:\Users\\.android\debug.keystore -list -v

Enter password: android

android is the default password. You can choose your own.

This command will generate your sha1 fingerprint something like below:

97:29:18:13:5D:B0:F2:44:02:27:61:A3:F7:1B:E4:FD:56:2D:A6:1B

On Create Oauth Client Id page, copy this sha1 code and paste in SHA-1 certificate fingerprint * field.

Open your project directory Our App-phonegap/config.xml file and copy widget id eg: com.phonegap.Our App

On create oauth cliend id page, enter com.phonegap.Our App in package name* field

Click create button.

On next page you will see oauth client id created message. Copy the oauth client id and reverse it in any text editor

If your client id is: 836754062314-hm43st16vdrfdgj54rcsi3rfvc5uwesm.apps.googleusercontent.com. Your reverse client id is: com.googleusercontent.apps.836754062314-hm43st16vdrfdgj54rcsi3rfvc5uwesm .

We have already added Google Plus Cordova Plugin in our code and included in our downloaded package. If you want the latest version you can install it from https://github.com/EddyVerbruggen/cordova-plugin-googleplus

In your project directory, edit below files.
- Our App-phonegap/plugins/fetch.json -> find cordova-plugin-googleplus Reversed Client Id: Replace it with your own reverse id
- Our App-phonegap/package.json find cordova-plugin-googleplus Reversed Client Id: Replace it with your own reverse id

If you follow above steps, you should have configured the google plus login plugin in your app successfully.

For any troubleshooting with this plugin, visit the official plugin page at https://github.com/EddyVerbruggen/cordova-plugin-googleplus.

Build Admin Panel

Now lets build our admin panel for hosting purpose.

All subsequent commands need to be run within the project's directory

Open your terminal and navigate to your project directory

cd Our App-reactjs-admin

By default as per the Our App-reactjs-admin/package.json file homepage variable, the build will assume that this app is hosted at /Our App-phonegap/admin/.

You can configure your homepage url in Our App-reactjs-admin/package.json file. Replace "homepage": "https://mydomain.com/Our App-phonegap/admin" with your desired app url.

Also make sure that proper values are configured in .env, .env.production files (as mentioned in above steps).

To Build your app for production, you need to run the "npm run build" command in your terminal window.


To change the directory to your own you need to replace the /Our App-phonegap/admin/ with your own directory various places mentioned above.


Following are the changes need to be done:

1. Open .env file and replace REACT_APP_BASEDIR=/Our App-phonegap/admin to REACT_APP_BASEDIR=/new/directory and REACT_APP_IMGDIR=/Our App-phonegap/admin to REACT_APP_IMGDIR=/new/directory. Note: Do not add "/" at the end of the directory path.

2. Open .env.production file and make similar changes as in .env file above.

3. Open package.json file and replace "homepage": "https://mydomain.com/Our App-phonegap/admin", to "homepage": "https://yourdomain/new/directory",. Note: Do not add "/" at the end of the directory path.

After changing to desired directory, you must run npm run build command to generate the production build ready for deployment.

This command will generate build folder in your project directory. This might take some time depending on the size of your project.

Once this directory is generated, you can upload this directory on your server via Ftp.

Visit the directory url on your server, you should see your react site up and running online.

Website Google Login

This website has an option to login user with a google account.

You need to create your own Oauth Client ID from Google Developer Console before making the google login work properly in this app.

First login in your google console developer here https://console.developers.google.com/

Create new project or choose existing one.

Then Go to credentials in side menu and click create credentials button. Select oauth client id

On next page, select application type as Web application.

Then, enter click add url in Authorised Redirect URL to add your website url for google login. According to our project this url is http://mydomain.com/Our App-phonegap/website/views/ui-social-google-login.php.

Click create button.

On next page you will see oauth client id created message. Copy the oauth client id and client secret key.

In your project directory, open - website/social/google-api.php file.
First replace MY-GOOGLE-CLIENT-ID with your own Client Id.
Then replace MY-GOOGLE-SECRET-KEY with your client Secret Key.

If you follow above steps, you should have configured the google login in your website successfully.

For any troubleshooting with this plugin, visit the official plugin page at https://github.com/googleapis/google-api-php-client.
